- Goal: rewrite Gentoo Portage `emerge` as idiomatic Rust in `diverge`.
- Reference checkout: `research/portage/`, ignored by git and treated as read-only research input.
- Primary reference entrypoint: `research/portage/bin/emerge`, which dispatches to `_emerge.main.emerge_main`.
- Core Portage behavior references: `research/portage/lib/_emerge/main.py`, `actions.py`, `create_depgraph_params.py`, `depgraph.py`, `Scheduler.py`, `Package.py`, `RootConfig.py`, `portage/package/ebuild/config.py`, `portage/eapi.py`, and `bin/ebuild.sh`.

- Testing spec: `docs/testing-spec.md` defines the Rust test-porting workflow, fixture rules, and coverage gate.
- Build status: all 8 planned phase milestones implemented end-to-end. 118 Rust parity tests + interop differential + e2e integration, ~86% line coverage, full gate green.
- Module map (single crate, layered): domain (`atom`, `version`, `dep`, `matching`), config (`config`, `util`, `profile`), repository (`dbapi`, `repository`, `manifest`), resolver (`depgraph`), interpretation (`cli`, `sets`, `update`), executor (`executor::{config_protect,merge,unmerge,phase}`), binpkg (`xpak`). `resolver.rs` is the original fixture resolver kept for `resolver_simple_parity`; `depgraph.rs` is the real resolver.

- Porting oracle: upstream Portage test files embed their own expected values, so ported Rust cases assert those literals (not invented ones). `use_reduce` does not yet port `opconvert`/`flat`/`is_src_uri`/`token_class` modes.
- Interop differential oracle: `tests/interop/portage_oracle.py` imports the REAL upstream Portage functions from `research/portage/lib` and emits canonical TSV; `tests/interop_differential.rs` diffs records against the Rust domain+config layers (vercmp, dep accessors, paren/use_reduce, check_required_use, cpv sort, varexpand, getconfig, match_from_list). It skips cleanly when `python3`/`research/portage` is absent or the oracle exits 77 (portage import failure). The harness prepends `research/portage/lib` to `PYTHONPATH` so it tests the checkout, not any system-installed portage. This is the anti-circularity oracle and it already caught a real `vercmp` suffix-ordering bug.
- vercmp suffix model now mirrors upstream `portage.versions`: `suffix_value = {alpha:-4, beta:-3, pre:-2, rc:-1, p:0}`, element-wise over multiple `_` suffixes, with a missing position padded as `("p", -1)` so `1 < 1_p0`.
- Required verification gate: `cargo fmt --all --check`, `cargo clippy --workspace --all-targets -- -D warnings`, `cargo test --workspace --all-targets`, and `cargo llvm-cov --workspace --all-targets --summary-only`.
- Coverage/linker note: `cargo llvm-cov` needs the `__llvm_covmap` ELF section, which a non-default global linker can drop (symptom: "no coverage data found" despite passing tests). `.cargo/config.toml` pins this crate to the `gcc`/`gcc`-flavor linker (last-wins over any global `linker-flavor=ld`) so the coverage gate is reproducible regardless of the developer's `~/.cargo/config.toml`. Verified root cause: a custom global linker built binaries with `__llvm_prf_*` but no `__llvm_covmap`.
- Coverage guardrail: new user-visible behavior should add or port tests first, then satisfy the llvm-cov gate before completion. Current total line coverage ~86%.
- Domain-layer EAPI gate detail: Portage's `_get_eapi_attrs(None)` sets `empty_groups_always_true=False` but `required_use_at_most_one_of=True`; `dep.rs` mirrors this so `|| ( )` is not implicitly satisfied under the permissive default.
